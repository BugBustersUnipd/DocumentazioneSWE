\section{Introduzione}

\subsection{Scopo del documento}
Il documento "Manuale utente" ha lo scopo di fornire agli utenti finali una guida completa e dettagliata sull'utilizzo dei Moduli AI Assistant generativo e AI Copilot per i consulenti del lavoro. 
Esso include informazioni su come installare, configurare e utilizzare le funzionalità principali della piattaforma, nonché indicazioni per la risoluzione di eventuali problemi comuni. 
L'obiettivo è garantire che gli utenti possano sfruttare appieno le potenzialità di Nexum per migliorare la comunicazione interna e la revisione e analisi di documenti all'interno delle loro organizzazioni.


\subsection{Scopo del prodotto\G}
Nexum è una piattaforma che supporta la gestione delle comunicazioni interne, la raccolta di feedback e la timbratura digitale del personale.
Il sistema consente di inviare comunicazioni ai dipendenti con tracciamento delle letture, creare e distribuire survey personalizzati e monitorarne i risultati tramite dashboard aggiornate in tempo reale. 
Inoltre, permette di effettuare la timbratura tramite badge o dispositivi mobili, applicando automaticamente le regole di controllo e aggregazione delle ore lavorate.

Le anagrafiche centralizzate, integrate con ruoli e permessi, consentono una gestione controllata degli accessi e garantiscono la coerenza delle informazioni tra i diversi moduli della piattaforma.

Questo manuale si occupa di un’applicazione stand-alone che in futuro verrà integrata con Nexum, che introduce funzionalità di assistente AI generativo per supportare l’utente nella scrittura di comunicazioni e un AI Co-Pilot dedicato ai Consulenti del Lavoro (CdL).
Questo modulo consente di caricare documenti, riconoscerne automaticamente la tipologia, estrarre le informazioni rilevanti e gestirne l’invio ai destinatari in modo massivo.

Per ulteriori dettagli sulle funzionalità e sul contesto del progetto, si rimanda alla documentazione di \href{https://bugbustersunipd.github.io/DocumentazioneSWE/RTB/ANALISI%20DEI%20REQUISITI/Analisi%20dei%20Requisiti.pdf}{Analisi dei Requisiti}.

\section{Requisiti}
Per poter utilizzare correttamente il prodotto\G, è necessario soddisfare alcuni requisiti minimi. 
Questi requisiti garantiscono che la piattaforma funzioni in modo efficiente e senza problemi, offrendo un'esperienza utente ottimale.


\subsection{Requisiti hardware}
Trattandosi di un'applicazione web, Nexum è accessibile da qualsiasi dispositivo dotato di un browser moderno e connesso a Internet.
Tuttavia, per garantire prestazioni ottimali, si consiglia di utilizzare un computer con le seguenti specifiche minime:
\begin{itemize}
    \item Processore: Intel Core i3 o equivalente
    \item RAM: 4 GB
    \item Spazio su disco: almeno 500 MB di spazio libero
    \item Connessione Internet: banda larga con velocità minima di 10 Mbps
\end{itemize}


\subsection{Requisiti software}
Per poter eseguire correttamente l'applicazione è necessario utilizzare un browser web aggiornato per garantire la compatibilità e la sicurezza (ininfluente il tipo di sistema operativo utilizzato).
I browser consigliati includono:
\begin{itemize}
    \item Google Chrome (versione 120 o superiore)
    \item Mozilla Firefox (versione 115 o superiore)
    \item Microsoft Edge (versione 120 o superiore)
    \item Safari (versione 17 o superiore)
\end{itemize}

Inoltre è necessario l'utilizzo di Docker per eseguire l'applicazione in un ambiente isolato e controllato, garantendo la compatibilità tra le diverse piattaforme e semplificando il processo di installazione e configurazione.
\begin{itemize}
    \item Versione minima di Docker: 19.03 con utilizzo di Docker Desktop 4.0.0
    \item Versione consigliata di Docker: 24 o superiore con utilizzo di Docker Desktop 4.3.0 o superiore
\end{itemize}


\section{Installazione}
Con il seguente procedimento è possibile installare e configurare l'applicazione stand-alone che introduce 
funzionalità di assistente AI generativo per supportare l’utente nella scrittura di comunicazioni e un AI Co-Pilot dedicato ai Consulenti del Lavoro (CdL).
Per un corretto funzionamento è necessario seguire ogni passaggio nell'ordine riportato.
Per motivi di sicurezza le API-key cambiano costantemente, quindi per poter usufruire correttamente dei moduli è necessario disporre di API-key aggiornate.

\subsection{Clonare la repository}
All'interno del proprio dispositivo:
\begin{itemize}
    \item Aprire il terminale e posizionarsi nella directory in cui si desidera clonare la repository.
    \item Eseguire il comando: \texttt{git clone <URL della repository>}
\end{itemize}

In questo modo si otterrà una copia locale del progetto\G, che potrà essere eseguita e modificata secondo le proprie esigenze.

\subsection{Configurare file env per le API-key di AWS}
Per poter utilizzare le funzionalità di AI generativa e di analisi documentale, è necessario configurare le API-key di AWS all'interno del file \texttt{.env} presente nella directory del progetto\G.
\begin{itemize}
    \item Aprire il file \texttt{.env} con un editor di testo.
    \item Inserire le API-key di AWS nei campi corrispondenti, seguendo il formato specificato nel file.
\end{itemize}
Assicurarsi di salvare le modifiche al file \texttt{.env} prima di procedere con l'esecuzione dell'applicazione.

\subsection{Inizializzazione applicazione tramite Docker}
Per avviare l'applicazione, è necessario utilizzare Docker per creare e gestire i container necessari al funzionamento della piattaforma.
\begin{itemize}
    \item Per costruire l'ambiente, eseguire il comando \texttt{docker-compose up}, che scarica l'immagine di Ruby e installa tutte le gemme necessarie definite nel Gemfile del progetto.
    \item Successivamente per costruire e popolare il database si utilizza il comando \texttt{docker compose run --rm web rails db:create db:migrate db:seed}, che esegue le migrazioni del database per creare le tabelle necessarie.
    \item Infine per avviare l'applicazione, eseguire il comando \texttt{docker compose up}, che avvia il server all'indirizzo \texttt{http://localhost:3000} e rende disponibile l'interfaccia utente all'indirizzo \texttt{http://localhost:4200}.
\end{itemize}




\section{Istruzioni per l'uso}

\subsection{Dashboard}
La dashboard è il punto di accesso ai moduli dell'applicazione. Da qui, gli utenti possono navigare tra le diverse funzionalità, visualizzare notifiche e accedere rapidamente 
alle varie sezioni. Nel contesto della nostra applicazione stand-alone, la dashboard funge da hub centrale per accedere sia all'AI-assistant generativo che all'AI Co-Pilot dedicato ai Consulenti del Lavoro (CdL).

\immagine[1\textwidth]{immagini/0_Dashboard.png}{Dashboard}{fig:Dashboard}

\subsubsection{Barra laterale}
Per permettere la navigazione tra i moduli, è presente una barra laterale che consente di accedere rapidamente alle sezioni principali dell'applicazione.

\immagine[0.3\textwidth]{immagini/0_BarraLaterale.png}{BarraLaterale}{fig:BarraLaterale}

\subsection{AI-assistant generativo}
Il modulo di AI-assistant generativo è progettato per supportare gli utenti nella creazione di comunicazioni interne.

\subsubsection{Generazione di contenuti}
La generazione dei contenuti avviene tramite l'inserimento di un prompt, la scelta di un tono e la scelta di uno stile da parte dell'utente. 
L'assistente Ai utilizza queste informazioni per generare un post formato da titolo, testo e immagine.

\immagine[1\textwidth]{immagini/1_GeneratoreDiContenuti.png}{GeneratoreDiContenuti}{fig:GeneratoreDiContenuti}

\subsubsubsection{Menù di scelta del tono}
Nel menù di scelta del tono, l'utente può cercare e selezionare tra diverse opzioni di tono (formale, informale, amichevole, professionale, ecc.). 
Inoltre ha anche la possibilità di inserire un tono personalizzato, specificando le caratteristiche desiderate (ad esempio, "tono motivazionale con un tocco di umorismo").
Oltre all'inserimento di un tono personalizzato, è presente anche un'opzione che consente all'utente di eliminare un tono reputato obsoleto o non soddisfacente.

\immagine[0.5\textwidth]{immagini/1_MenuTono.png}{MenuTono}{fig:MenuTono}


\subsubsubsection{Menù di scelta dello stile}
Nel menù di scelta dello stile, l'utente può cercare e selezionare tra diversi stili di scrittura (conciso, dettagliato, persuasivo, ecc.).
Anche in questo caso, è possibile inserire uno stile personalizzato, indicando le preferenze specifiche (ad esempio, "stile narrativo con esempi concreti").
Oltre all'inserimento di uno stile personalizzato, è presente anche un'opzione che consente all'utente di eliminare uno stile reputato obsoleto o non soddisfacente.

\immagine[0.5\textwidth]{immagini/1_MenuStile.png}{MenuStile}{fig:MenuStile}


\subsubsubsection{Casella di testo per inserimento del prompt}
La casella di testo per l'inserimento del prompt consente all'utente di fornire indicazioni specifiche su ciò che desidera comunicare.
Il prompt può includere informazioni chiave, obiettivi della comunicazione o qualsiasi dettaglio che l'utente ritiene importante per guidare la generazione del contenuto da parte dell'assistente AI.

\immagine[1\textwidth]{immagini/1_CasellaInserimentoPrompt.png}{CasellaPrompt}{fig:CasellaPrompt}

\subsubsubsection{Tasto di generazione}
Dopo aver compilato i campi necessari (prompt, tono e stile), l'utente può cliccare sul tasto di generazione per avviare il processo di creazione del contenuto.

\immagine[0.4\textwidth]{immagini/1_TastoGenerazioneContenuto.png}{TastoGenerazione}{fig:TastoGenerazione}


\subsubsection{Anteprima del post generato}
A seguito della generazione, viene visualizzata un'anteprima del post generato, che potrà essere modificato o confermato dall'utente prima di essere pubblicato o salvato. Inoltre l'utente 
dovrà anche valutare la qualità del contenuto generato.

\immagine[1\textwidth]{immagini/1_AnteprimaGenerazione.png}{AnteprimaGenerazione}{fig:AnteprimaGenerazione}

\subsubsubsection{Immagine generata}
L'immagine generata viene mostrata come primo elemento dell'anteprima del post, fornendo un supporto visivo che arricchisce il contenuto testuale e rende la comunicazione più accattivante e coinvolgente per i destinatari.  

\immagine[1\textwidth]{immagini/1_ImmagineGenerazione.png}{ImmagineGenerazione}{fig:ImmagineGenerazione}


\subsubsubsection{Titolo generato}
All'interno dell'anteprima del post, il titolo generato viene visualizzato in modo prominente, evidenziando l'argomento principale della comunicazione ai destinatari.

\immagine[0.8\textwidth]{immagini/1_TitoloGenerazione.png}{TitoloGenerazione}{fig:TitoloGenerazione}


\subsubsubsection{Testo generato}
Il testo viene mostrato sotto l'immagine e il titolo, fornendo il corpo principale della comunicazione che l'utente ha richiesto all'assistente AI di generare.

\immagine[1\textwidth]{immagini/1_TestoGenerazione.png}{TestoGenerazione}{fig:TestoGenerazione}


\subsubsubsection{Valutazione del post generato}
Dopo aver visualizzato l'anteprima del post generato, l'utente ha la possibilità di valutare la qualità del contenuto attraverso un sistema basato su un punteggio da 1 a 5 stelle.

\immagine[0.4\textwidth]{immagini/1_Valutazione.png}{Valutazione}{fig:Valutazione}


\subsubsubsection{Panoramica parametri di generazione}
Per poter associare più facilmente il risultato generato ai parametri di generazione, è presente una sezione che mostra in modo chiaro e sintetico i parametri utilizzati per la generazione del contenuto (il tono, lo stile e il prompt inserito dall'utente).

\immagine[1\textwidth]{immagini/1_PanoramicaParametri.png}{PanoramicaParametri}{fig:PanoramicaParametri}

\subsubsubsection{Tasto accesso modalità modifica}
Per attivare le funzionalità di modifica del post generato, è presente un tasto che consente all'utente di accedere alla modalità di modifica, dove potrà apportare le modifiche desiderate al titolo, al testo e all'immagine generati.

\immagine[0.4\textwidth]{immagini/1_TastoModalitaModifica.png}{TastoModalitaModifica}{fig:TastoModalitaModifica}

\immagine[0.6\textwidth]{immagini/1_OperazioniGenerazione.png}{OperazioniGenerazione}{fig:OperazioniGenerazione}


\subsubsection{Modifica delle generazioni}
Entrando nella modalità di modifica, l'utente può apportare modifiche al titolo, al testo e all'immagine generati, personalizzando ulteriormente il contenuto per adattarlo meglio alle proprie esigenze comunicative.

\immagine[1\textwidth]{immagini/1_ModalitaModificaGenerazione.png}{ModalitaModificaGenerazione}{fig:ModalitaModificaGenerazione}

\subsubsubsection{Modifica immagine generata}
Nella modalità di modifica, l'utente ha la possibilità di sostituire l'immagine generata dall'AI con una nuova, avviando una rigenerazione sulla base dei parametri originali o di un prompt aggiornato.

\immagine[0.6\textwidth]{immagini/1_ModificaImmagine.png}{ModificaImmagine}{fig:ModificaImmagine}

\subsubsubsection{Modifica titolo generato}
Nella modalità di modifica, l'utente ha la possibilità di modificare il titolo generato, adattandolo meglio al contesto comunicativo desiderato.

\immagine[0.8\textwidth]{immagini/1_ModificaTitolo.png}{ModificaTitolo}{fig:ModificaTitolo}


\subsubsubsection{Modifica testo generato}
Nella modalità di modifica, l'utente ha la possibilità di modificare il testo generato, personalizzandolo ulteriormente per renderlo più efficace e adatto alle proprie esigenze comunicative.

\immagine[0.8\textwidth]{immagini/1_ModificaTesto.png}{ModificaTesto}{fig:ModificaTesto}


\subsubsubsection{Salvataggio o annullamento modifiche alla generazione}
La modifica può essere salvata per aggiornare il post generato con le modifiche apportate, oppure annullata per tornare alla versione originale del contenuto generato.

\immagine[0.8\textwidth]{immagini/1_GestioneModifiche.png}{GestioneModifiche}{fig:GestioneModifiche}


\subsection{Storico delle generazioni}
Lo storico delle generazioni consente all'utente di visualizzare tutte le generazioni precedenti, con la possibilità di accedere ai dettagli di ciascuna generazione, come i parametri utilizzati, la valutazione e le eventuali modifiche apportate.

\subsubsection{Consultazione storico delle generazioni}
Lo storico delle generazioni è una sezione composta da una tabella contenente le generazioni e vari sistemi di input che permettono di effettuare operazioni su questa tabella con fini di gestione o di analisi.

\subsubsubsection{Filtraggio e ricerca nello storico}
All'interno dello storico delle generazioni, è presente una funzionalità di filtraggio e ricerca che consente all'utente di trovare facilmente specifiche generazioni passate in base a criteri come data, valutazione, parole chiave, tono o stile utilizzato.

\subsubsubsection{Visualizzazione tabella generazioni}
Le generazioni passate vengono visualizzate in una tabella che mostra informazioni chiave come la data di creazione, il titolo del post generato, il tono e lo stile utilizzati, la valutazione assegnata, il testo generato e il prompt utilizzato.

\subsubsubsection{Azioni su generazioni passate}
Per ogni generazione passata visualizzata nella tabella, l'utente ha la possibilità di eseguire azioni come riutilizzare, duplicare o eliminare la generazione dallo storico.
Riutilizzare una generazione consiste nell'effettiva generazione di un nuovo post con gli stessi campi, mentre la duplicazione riempie i campi permettendo di apportare modifiche prima di effettuare la nuova generazione.

\subsubsubsection{Scorrimento tabella storico}
Utilizzando frecce a fondo tabella l'utente può scorrere tra le pagine dello storico delle generazioni, visualizzando un numero limitato di generazioni per pagina per facilitare la consultazione e la gestione dello storico.



\subsection{AI Co-Pilot per i Consulenti del Lavoro}
L'AI Co-Pilot dedicato ai Consulenti del Lavoro (CdL) è progettato per supportare i professionisti nella gestione e nell'analisi dei documenti, facilitando il processo di revisione e invio dei documenti ai destinatari.

\subsubsection{Analisi documenti}
Il modulo di analisi dei documenti consente ai CdL di caricare documenti in vari formati (PDF, Word, ecc.) e di riconoscerne automaticamente la tipologia (ad esempio, busta paga, contratto di lavoro, comunicazione interna, ecc.).
Per effettuare un'analisi é necessario inserire il file che si ha intenzione di analizzare ed eventualmente dati conosciuti, facilitando così il processo di riconoscimento e estrazione delle informazioni rilevanti.

\subsubsubsection{Caricamento documento}
Tramite questa funzione, l'utente può selezionare e caricare uno o più documenti all'interno del sistema. L'infrastruttura verifica l'integrità e la validità del formato del file per prepararlo all'analisi tramite intelligenza artificiale.

\subsubsubsection{Inserimento dati conosciuti}
Per guidare il modello di AI e aumentare la precisione dell'estrazione (confidence), l'utente ha la facoltà di inserire manualmente alcuni metadati ancor prima di lanciare l'analisi. Tra questi figurano la selezione della categoria documentale, l'inserimento dell'azienda cliente, l'indicazione del mese/anno di competenza e lo specifico reparto aziendale.


\subsubsection{Visualizzazione risultati analisi documenti}
Dopo l'analisi, i risultati vengono visualizzati in modo chiaro e organizzato, evidenziando le informazioni estratte dal documento e fornendo un quadro completo del contenuto analizzato.
Questa visualizzazione consente al CdL di comprendere rapidamente le informazioni chiave del documento e di prendere decisioni informate su come procedere con la gestione del documento stesso.
 
\subsubsubsection{Visualizzazione tabella riepilogativa documenti analizzati}
Il sistema presenta l'elenco dei documenti analizzati in formato tabellare. Per ogni elemento, vengono esposti i dati di sintesi estratti dall'AI, quali l'azienda, il periodo di competenza, la tipologia del file, la lingua, il numero di pagine e l'identificativo originale, permettendo un controllo rapido del flusso elaborato.

\subsubsubsection{Visualizzazione stato analisi documenti}
Questa funzionalità permette all'utente di monitorare l'avanzamento e la fase del ciclo di vita in cui si trova il documento. Attraverso etichette testuali o indicatori visivi (es. "Da validare", "Pronto per l'invio", "Inviato", "Errore"), è possibile comprendere immediatamente se il documento necessita di intervento umano o se l'elaborazione è andata a buon fine.

\subsubsubsection{Applicazione filtro di ricerca su documenti}
Per agevolare la gestione di archivi voluminosi, il sistema mette a disposizione uno strumento di filtraggio dinamico. L'utente può raffinare la lista dei risultati applicando filtri per tipologia documentale, mese/anno di competenza, livello di confidenza dell'AI, documento originale o stato di elaborazione.

\subsubsubsection{Modifica del risultato dell'analisi documenti}
In ottica "Human-in-the-loop", se l'AI dovesse classificare un documento in modo impreciso o associare dati non corretti, l'operatore può intervenire manualmente per modificare la tipologia del documento o correggerne gli attributi. Questa operazione induce il sistema a ricalcolare e aggiornare la percentuale di confidenza.

\subsubsubsection{Eliminazione del risultato dell'analisi documenti}
Qualora un documento sia stato caricato per errore, risulti duplicato o non debba più far parte del flusso di invio, l'utente può rimuovere il documento e i relativi risultati dell'analisi dal database e dall'interfaccia corrente.


\subsubsection{Riconoscimento destinatari con creazione liste di invio}
Il sistema è in grado di riconoscere automaticamente i destinatari associati al documento analizzato, creando liste di invio basate sulle informazioni estratte.
Questa funzionalità semplifica il processo di invio dei documenti, consentendo al CdL di selezionare facilmente i destinatari appropriati e di gestire le comunicazioni in modo più efficiente.

\subsubsubsection{Visualizzazione risultati analisi destinatari}
Il sistema mostra, a seguito dell'analisi documentale (e dell'eventuale split dei documenti massivi), l'elenco dei destinatari individuati. I dati mostrati includono nome, cognome, codice fiscale, numero di matricola e reparto di afferenza, garantendo la verifica dell'identità.

\subsubsubsection{Visualizzazione stato analisi destinatari}
Permette di visualizzare se i dati anagrafici estratti dall'AI per ciascun destinatario sono considerati affidabili, se necessitano di validazione manuale o se non è stato possibile riconoscere alcun destinatario, richiedendo così l'attenzione dell'operatore.

\subsubsubsection{Applicazione filtro di ricerca su destinatari}
Per rintracciare rapidamente i destinatari all'interno di invii massivi, l'utente può utilizzare barre di ricerca e filtri mirati. La lista può essere filtrata per numero di matricola, per reparto aziendale o in base allo specifico codice del documento associato.

\subsubsubsection{Modifica del risultato dell'analisi destinatari}
Nel caso l'algoritmo associ un destinatario errato a un documento o non riesca a dedurlo, l'operatore può intervenire per cercare e selezionare il destinatario corretto. L'associazione può avvenire tramite l'inserimento manuale dell'indirizzo email o del numero di matricola.

\subsubsubsection{Eliminazione del risultato dell'analisi destinatari}
Garantisce la possibilità di dissociare un destinatario da un documento o di rimuoverlo dalla lista di distribuzione creata, evitando che riceva comunicazioni non di sua competenza.


\subsubsection{Visualizzazione anteprima e correzione documenti}
A seguito dell'analisi, il sistema estrae le informazioni rilevanti dal documento e le presenta in modo chiaro e organizzato, consentendo al CdL di verificare l'accuratezza dei dati estratti e di apportare eventuali correzioni o integrazioni prima di procedere con l'invio del documento ai destinatari.

\subsubsubsection{Visualizzazione anteprima documento}
Il sistema mostra a schermo l'anteprima visiva del file sorgente (es. il PDF o l'immagine caricata). Questa funzione è essenziale per consentire all'operatore di leggere il documento originale e confrontarlo direttamente e in tempo reale con le estrazioni effettuate dall'AI.

\subsubsubsection{Visualizzazione informazioni documento}
In una sezione adiacente all'anteprima, il sistema raccoglie tutti i metadati specifici del singolo file, quali il codice univoco interno, la data di caricamento e redazione, la causale individuata e il grado di affidabilità dell'estrazione (confidence score).

\subsubsubsection{Visualizzazione destinatari documento}
Visualizza nel dettaglio i soggetti a cui il singolo documento in anteprima è indirizzato. In presenza di liste lunghe, questa sezione può presentare un riepilogo rapido e mostrare l'appartenenza a specifiche liste di distribuzione.

\subsubsubsection{Visualizzazione dettagli analisi}
Permette di consultare le metriche tecniche del processo elaborativo eseguito dall'AI, evidenziando le criticità (es. confidenza del dato inferiore alla soglia di sicurezza) e consultando l'audit log, che riporta cronologicamente tutte le modifiche e i cambi di stato del documento nel tempo.

\subsubsubsection{Esecuzione operazioni su documento}
Fornisce i comandi d'azione diretti sul singolo file in fase di revisione. L'utente può confermare (validare) le estrazioni dell'AI, correggere campi di testo, scaricare il documento originale o procedere alla sua eliminazione in caso di anomalie insormontabili.

\subsubsubsection{Esecuzione operazioni su destinatari}
Consente le modifiche mirate alle anagrafiche dei soggetti associati al documento visualizzato in anteprima, facilitando l'operazione di sgancio o aggiornamento manuale del destinatario finale.


\subsubsection{Invio documenti}
Quando si ritiene che il documento sia pronto per essere inviato (con una confidenza adeguata), l'utente può procedere con la creazione di un messaggio, l'eventuale inserimento di allegati ed infine l'invio del documento ai destinatari.

\subsubsubsection{Scrittura messaggio}
Questa funzionalità mette a disposizione un editor di testo per la composizione della comunicazione che verrà recapitata ai destinatari. L'utente può valorizzare manualmente il campo "Oggetto" e inserire il corpo testuale desiderato.

\subsubsubsection{Inserimento allegati}
Oltre al documento primario processato (es. la busta paga o la certificazione), il sistema permette l'upload e l'aggiunta di ulteriori file (es. moduli informativi o comunicazioni aziendali) che verranno spediti congiuntamente al messaggio.

\subsubsubsection{Applicazione template}
Per ottimizzare i tempi, l'utente può decidere di non scrivere il messaggio da zero, bensì caricare un modello (template) precostituito. Questo andrà a sovrascrivere o precompilare i campi dell'oggetto e del corpo del messaggio con formule testuali standard.

\subsubsubsection{Operazioni su template esistenti}
L'utente ha accesso a una gestione completa dei modelli di comunicazione: può visualizzare la lista dei template salvati, creare un nuovo template a partire dal messaggio che sta scrivendo, oltre a modificare e cancellare definitivamente quelli divenuti obsoleti.

\subsubsubsection{Programmazione invio documento}
Se non vi è urgenza o si desidera sincronizzare l'inoltro dei documenti con specifiche scadenze aziendali, il sistema permette di schedulare l'operazione. Tramite un apposito selettore (date/time picker), l'utente definisce l'esatta data e ora di invio, e lo stato passerà a "Pianificato".

\subsubsubsection{Invio immediato documento}
Terminata la verifica e la preparazione della comunicazione, questa funzionalità fa partire istantaneamente il processo di spedizione verso tutti i destinatari associati al documento, fornendo all'utente un feedback visivo a conferma del successo dell'operazione.




\subsection{Storico documenti}
L'utente ha a disposizione uno storico dei documenti analizzati, visualizzandone i dettagli e il loro stato.

\subsubsection{Consultazione storico documenti}
Lo storico dei documenti consente al CdL di visualizzare tutte le analisi documentali precedenti, con la possibilità di accedere ai dettagli di ciascun documento, come la data di analisi, la tipologia del documento, i destinatari riconosciuti, le informazioni estratte e lo stato dell'invio.


\subsection{Analytics e monitoraggio trasversale}
L'utente ha a disposizione una sezione dedicata all'analytics e al monitoraggio trasversale, che consente di visualizzare metriche e statistiche relative all'utilizzo dell'AI-assistant generativo e dell'AI Co-Pilot per i Consulenti del Lavoro (CdL), fornendo insights utili per valutare l'efficacia delle funzionalità offerte dalla piattaforma e per identificare eventuali aree di miglioramento.

\subsubsection{Analytics e monitoraggio trasversale per l'AI-assistant generativo}
L'utente ha la possibilità di accedere a una sezione dedicata all'AI-assistant generativo, con un elenco di metriche e statistiche che forniscono insights sull'utilizzo e sulle prestazioni dell'assistente AI, consentendo di valutare l'efficacia delle generazioni, la soddisfazione degli utenti e l'impatto complessivo sulla comunicazione interna.
Tramite lo studio di queste metriche, è possibile identificare eventuali aree di miglioramento per ottimizzare ulteriormente le funzionalità e garantire un'esperienza utente sempre più soddisfacente.

\subsubsubsection{Numero di generazioni effettuate}
Questa metrica permette all'utente di visualizzare il volume totale di richieste, o prompt, inviate all'AI nel periodo temporale selezionato. Il sistema interroga il database storico e conteggia il numero totale di interazioni avviate, mostrando il dato numerico all'interno della dashboard.

\subsubsubsection{Valutazione media delle generazioni}
Tramite questo indicatore, l'utente può valutare la qualità percepita delle risposte fornite dall'AI Assistant. Il sistema aggrega i feedback ricevuti (tramite stelle) e calcola la media aritmetica dei voti, mostrando un valore che permette di comprendere il livello di soddisfazione dell'utenza.

\subsubsubsection{Numero di rigenerazioni per prompt}
Questa statistica mostra quante volte è stata utilizzata dagli utenti la funzione "Rigenera risposta". Il sistema conta le occorrenze in cui viene richiesta una nuova versione di un messaggio già generato: si tratta di un dato utile per capire se la prima risposta dell'AI risulta spesso insoddisfacente.

\subsubsubsection{Utilizzo toni e stili}
Questa sezione evidenzia le preferenze degli utenti riguardo ai formati e ai registri di comunicazione. Il sistema raggruppa i prompt generati in base agli attributi "Tono" (es. Formale, Empatico) e "Stile" (es. Sintetico, Dettagliato), calcolandone la frequenza di utilizzo. I dati vengono visualizzati tramite grafici o classifiche che mettono in risalto i toni e gli stili più richiesti.

\subsubsubsection{Filtraggio per periodo temporale AI-assistant generativo}
L'utente ha la possibilità di filtrare tutte le metriche e statistiche relative all'AI-assistant generativo in base a un intervallo di tempo specifico (ad esempio, ultimi 7 giorni, ultimo mese, ultimo trimestre, ecc.). Il sistema applica il filtro temporale a tutte le metriche visualizzate, consentendo di analizzare l'andamento delle generazioni e delle preferenze degli utenti in periodi specifici, facilitando così l'identificazione di trend o variazioni significative nel tempo.

\subsubsection{Analytics e monitoraggio trasversale per l'AI Co-Pilot per i Consulenti del Lavoro}
L'utente ha la possibilità di accedere a una sezione dedicata all'AI Co-Pilot per i Consulenti del Lavoro, con un elenco di metriche e statistiche che forniscono insights sull'utilizzo e sulle prestazioni del Co-Pilot, consentendo di valutare l'efficacia delle generazioni, la soddisfazione degli utenti e l'impatto complessivo sulla comunicazione interna.
Tramite lo studio di queste metriche, è possibile identificare eventuali aree di miglioramento per ottimizzare ulteriormente le funzionalità e garantire un'esperienza utente sempre più soddisfacente.

\subsubsubsection{Percentuale confidenza media}
Questa metrica mostra il punteggio medio di affidabilità (confidence score) che l'AI ha attribuito alle proprie estrazioni di dati nel periodo selezionato. Il sistema calcola la media (pesata o aritmetica) dei punteggi associati a tutti i documenti elaborati, indicando inoltre se tale valore è in linea, superiore o inferiore rispetto ai trend storici.

\subsubsubsection{Percentuale intervento umano per correzione}
Questo indicatore visualizza la frequenza con cui gli operatori umani ("Human-in-the-loop") devono intervenire per correggere i dati estratti dall'AI prima del salvataggio. Analizzando i log di audit, il sistema confronta il numero di documenti che hanno subito modifiche manuali con il volume totale dei documenti processati, esprimendo il risultato in percentuale.

\subsubsubsection{Accuratezza mapping}
Questa statistica rappresenta la precisione con cui l'AI associa i dati individuati nei documenti ai campi corretti all'interno del database. Il sistema valuta la coerenza dei dati e calcola un indice di accuratezza strutturale, permettendo di identificare se l'AI sta fallendo nel riconoscere specifiche tipologie di documento o determinati layout.

\subsubsubsection{Tempo medio di analisi documenti}
Questa metrica indica il tempo medio impiegato dal sistema per processare ed elaborare un documento. Il calcolo viene effettuato basandosi sui timestamp di inizio (upload) e di fine (disponibilità dei dati), restituendo un valore fondamentale per monitorare le performance dell'infrastruttura e la latenza del servizio.

\subsubsubsection{Filtraggio per periodo temporale AI Co-Pilot}
L'utente ha la possibilità di filtrare tutte le metriche e statistiche relative all'AI Co-Pilot in base a un intervallo di tempo specifico (ad esempio, ultimi 7 giorni, ultimo mese, ultimo trimestre, ecc.). Il sistema applica il filtro temporale a tutte le metriche visualizzate, consentendo di analizzare l'andamento delle analisi.

\section{Test}
Per garantire la qualità e l'affidabilità del prodotto, è stato implementato un processo di testing completo che include test unitari, test di integrazione e test end-to-end.
Il processo di testing è stato eseguito in diverse fasi del ciclo di sviluppo, consentendo di identificare e risolvere tempestivamente eventuali bug o problemi di funzionalità.

\subsection{Test Backend}
Il focus dei test backend è stato principalmente sulla verifica della corretta implementazione delle funzionalità, sulla gestione degli errori e sulla sicurezza dell'applicazione. Sono stati eseguiti test unitari per verificare il funzionamento di singole componenti e test di integrazione per assicurare che le diverse parti del sistema interagiscano correttamente tra loro.
Per generare e avviare i test backend, è stato utilizzato il framework di testing integrato in Ruby on Rails, che consente di scrivere test in modo semplice e intuitivo. I test sono stati organizzati in diverse categorie.

\subsubsection{Test di unità}
Per avviare i test unitari bisogna utilizzare il comando \texttt{docker compose run --rm web rails test:units}, che esegue tutti i test unitari presenti nella directory \texttt{test/unit} del progetto.
 
\subsubsection{Test di integrazione}
Per avviare i test di integrazione bisogna utilizzare il comando \texttt{docker compose run --rm web rails test:integration}, che esegue tutti i test di integrazione presenti nella directory \texttt{test/integration} del progetto.
